## Questions for Meeting

How to deal with `mc.cores` option and `cores` arguments? 

Update `loo_cores()` validation helper, and change defaults from `cores = getOption("mc.cores", 1)` to `cores = 1`, or to `NULL` (i.e. default to serial or parallel execution?)?

**Old:**

```r
# Check how many cores to use and throw deprecation warning if loo.cores is used
loo_cores <- function(cores) {
    loo_cores_op <- getOption("loo.cores", NA)
  if (!is.na(loo_cores_op) && (loo_cores_op != cores)) {
      cores <- loo_cores_op
    warning("'loo.cores' is deprecated, please use 'mc.cores' or pass 'cores' explicitly.",
            call. = FALSE)
  }
  return(cores)
}
```

**Rough Proposal:**

```r
# Check how many cores to use and throw deprecation warning if loo.cores is used
loo_cores <- function(cores) {
  # TODO: link parallelization vignette
  if (!is.na(getOption("loo.cores", NA))) {
    warning(
      "'loo.cores' is deprecated, please remove 'loo.cores' and use mirai::daemons() for parallelization. See the parallelization vignette for more details.",
      call. = FALSE
    )
  }
  if (is.null(cores)) {
    if (!requireNamespace("mirai", quietly = TRUE)) {
      stop(
        "`cores = NULL` requires the 'mirai' package. Please install `mirai`, call `mirai::daemons(n)`, and then call loo with `cores = NULL`. See the parallelization vignette for more details.",
        call. = FALSE
      )
    }
    # mirai_map() when no daemons are set says:
    #
    #> Error in `mirai::mirai_map()`:
    #> ! No daemons set.
    #> Use e.g. mirai::daemons(6) to set 6 local daemons.

    # but we could also do our own check
    # if (!mirai::daemons_set()) {
    #   stop(
    #     "`cores = NULL` requires `mirai` daemons to be set. Please call `mirai::daemons(n)` before calling loo with `cores = NULL`. See the parallelization vignette for more details.",
    #     call. = FALSE
    #   )
    # }

    return(NULL)
  }

  if (cores != 1) {
    stop(
      "`cores` no longer controls parallelization. Please call mirai::daemons(",
      cores,
      ") before calling loo, and don't pass `cores`. See the parallelization vignette for more details.",
      call. = FALSE
    )
  }

  cores
}
```

New `loo_cores()` maintains contract that `cores` should be `1` for serial execution, and `NULL` for parallel execution.
