# `loo` Parallelization

General documentation for Stan/Numfocus GSOC 2026 project: [parallelizing `loo`](https://www.visruth.com/blog/20260516-01/). 

`experiments` contain code and are more empirical; `notes` is less empirical and contains some design decisions and things to keep in mind, or things we want to implement. One should view the built markdown results instead of the Quarto docs--they should be easy to see in any viewer which supports GitHub flavoured markdown (viz. the GitHub website).
